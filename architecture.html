LLM-Callable Tools — PaymentDataTools

Six @Tool-annotated methods registered with the ChatClient. gpt-oss:20b decides which to call based on the user's question — Spring AI executes them and feeds results back automatically. All return mock data suitable for sales demos.

getRecentTransactions
(int limit) → List<Map>
Returns the most recent payment transactions with sender, receiver, amount, status, and risk score.
getPaymentStatistics
() → Map
24-hour KPI summary: total transactions, total volume, flagged count, fraud rate, average amount.
getFraudAlerts
(int limit) → List<Map>
Current fraud alerts with rule name, amount, risk score, sender account, and triggered time.
explainFraudDetectionRules
() → String
Documents the four fraud rules: HIGH_AMOUNT, RAPID_SUCCESSION, ROUND_NUMBER, HIGH_RISK_ACCOUNT.
getAccountBalances
() → List<Map>
Current balances for all six fictional demo banks: Albion, Meridian, Crestfield, Harrington, Caledonian, Vantage.
generateFraudNarrative
(String txnRef) → String
Generates a detailed investigation narrative for a transaction — timeline, risk factors, triggered rules, recommended action.
Tool annotation — PaymentDataTools.java
// Spring AI 1.0 — method-level @Tool annotation @Tool("Get the N most recent payment transactions with risk scores") public List<Map<String, Object>> getRecentTransactions(int limit) { return List.of( Map.of("txnId", "PF-20261007", "amount", "£125,000.00", "sender", "Vantage Bank PLC", "riskScore", 82, ...), ... ).stream().limit(limit).toList(); }

Chat Request — Full Lifecycle

1
User sends message
React calls POST /api/chat with { message, sessionId }. If no sessionId is provided (first message), the server generates one and returns it for subsequent requests.
REST JSONsessionId
2
Conversation history prepended
MessageChatMemoryAdvisor looks up the sessionId in InMemoryChatMemory and prepends prior messages before the prompt is sent to the LLM. Each session has independent history.
InMemoryChatMemoryMessageChatMemoryAdvisor
3
LLM call #1 — gpt-oss:20b decides which tools to use
Spring AI sends the prompt (with history, system message, and tool schemas) to gpt-oss:20b. The model responds with tool_calls JSON indicating which @Tool methods to invoke.
gpt-oss:20btool_calls JSON
4
@Tool methods executed by Spring AI
Spring AI reflectively calls the matching method(s) on PaymentDataTools. Results are serialised to JSON and added to the message history as tool result messages.
Java reflectionPaymentDataTools
5
LLM call #2 — synthesise response from tool data
Spring AI sends another request to gpt-oss:20b with the tool results in context. The model synthesises a natural-language answer grounded in the live data — no hallucinated figures.
gpt-oss:20bData-grounded response
✓
Response returned to browser
{ response: "...", sessionId: "..." } returned as JSON. React appends the assistant message. The sessionId is kept in state for the next request.
JSON responseReact state update

Build & Deployment

1
Build React frontend
npm ci && npm run build inside frontend/. Vite outputs the bundle to backend/src/main/resources/static/. Spring Boot serves it at /.
Vite 5React 18Tailwind CSS 3
2
Package Spring Boot jar
mvn clean package -DskipTests inside backend/. Produces backend/target/payflow-ai.jar with Java classes, Spring AI dependencies, and embedded React assets.
Spring Boot 3.5Spring AI 1.0JDK 21
3
Create AI service instance (one-time)
cf create-service ai-models chat-and-tools-model payflow-ai-chat-tools. Listed in manifest.yml services: — CF injects credentials into VCAP_SERVICES on push. No cf set-env for credentials. Nothing secret in git.
cf create-serviceVCAP_SERVICESNo credentials in git
4
cf push
CF CLI reads manifest.yml, uploads the jar, stages with java_buildpack_offline (JDK 21), binds payflow-ai-chat-tools, assigns PORT, and starts the app. GenAiVcapPostProcessor maps VCAP_SERVICES to Spring AI properties before any beans are created.
java_buildpack_offlineJDK 211G memory
✓
App is live
React SPA at /. Chat at POST /api/chat. Health at /actuator/health. No CORS — frontend and backend share one CF route.
Single routeNo CORS

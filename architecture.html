LLM-Callable Tools — PaymentDataTools

Six @Tool-annotated methods registered with the ChatClient. The LLM decides which to call based on the user's question — Spring AI executes them and feeds results back automatically. All return mock data suitable for sales demos.

getRecentTransactions
(int limit) → List<Map>
Returns the most recent payment transactions with sender, receiver, amount, status, and risk score. Use limit to control how many to show.
getPaymentStatistics
() → Map
24-hour KPI summary: total transactions, total volume, flagged count, fraud rate, average amount, and processing time.
getFraudAlerts
(int limit) → List<Map>
Current fraud alerts with rule name, amount, risk score, sender account, and triggered time. Sorted by risk descending.
explainFraudDetectionRules
() → String
Documents the four fraud rules: HIGH_AMOUNT (>£50k), RAPID_SUCCESSION (>3 txns/60s), ROUND_NUMBER (exact £1k+ multiples), HIGH_RISK_ACCOUNT (flagged account history).
getAccountBalances
() → List<Map>
Current balances for all six fictional demo banks: Albion, Meridian, Crestfield, Harrington, Caledonian, Vantage.
generateFraudNarrative
(String txnRef) → String
Generates a detailed investigation narrative for a transaction reference — timeline, risk factors, triggered rules, and recommended action.
Tool annotation example — PaymentDataTools.java
// Spring AI 1.0 — method-level @Tool annotation @Tool("Get the N most recent payment transactions with risk scores") public List<Map<String, Object>> getRecentTransactions(int limit) { // returns mock data — no DB required return List.of( Map.of("txnId", "PF-20261007", "amount", "£125,000.00", "sender", "Vantage Bank PLC", "riskScore", 82, ...), ... ).stream().limit(limit).toList(); }

Chat Request — Full Lifecycle

1
User sends message
React calls POST /api/chat with { message, sessionId }. If no sessionId is provided, the server generates one (UUID) and returns it in the response for the browser to reuse in subsequent messages.
REST JSONsessionId
2
Conversation history prepended
MessageChatMemoryAdvisor looks up the sessionId in InMemoryChatMemory and prepends the previous N messages before the prompt is sent to the LLM. Each session maintains independent history.
InMemoryChatMemoryMessageChatMemoryAdvisor
3
LLM call #1 — gpt-oss:20b decides to use tools
Spring AI sends the prompt (with history and system message) to gpt-oss:20b. The LLM's tool-calling response includes one or more tool invocations. Spring AI parses the response and calls the appropriate @Tool methods automatically.
gpt-oss:20btool_calls JSON
4
@Tool methods executed
Spring AI reflectively calls the matching method(s) on PaymentDataTools (e.g. getRecentTransactions(5), getFraudAlerts(3)). Results are serialised to JSON strings and added to the message history as tool result messages.
Java reflectionPaymentDataTools
5
LLM call #2 — synthesise the response
Spring AI sends another request to gpt-oss:20b with the tool results in context. The LLM synthesises a natural-language answer using the live data. The final string content is returned.
gpt-oss:20bData-grounded response
✓
Response returned
{ response: "...", sessionId: "..." } returned to the browser. React appends the assistant message to the chat history. The sessionId is stored in component state for the next request.
JSON responseReact state update

Semantic Search — Embedding Flow

Startup — seed the vector store
S
@PostConstruct — 20 documents
SemanticSearchService.seedVectorStore() creates 20 Document objects covering key fraud patterns and transaction types.
↓
EmbeddingModel embeds each doc
vectorStore.add(docs) calls nomic-embed-text-v2-moe for each document, stores float[] vectors in memory.
✓
Store ready
20 vectors in memory. App serves search requests immediately — no warmup latency after startup.
Runtime — handle a search query
1
GET /api/search?q=...
Browser sends query string and optional topK (default 5). SearchController delegates to SemanticSearchService.search().
2
Query embedded
SimpleVectorStore.similaritySearch() calls nomic-embed-text-v2-moe to embed the query into the same vector space as the documents.
3
Cosine similarity + rank
SimpleVectorStore computes dot-product similarity against all 20 stored vectors, returns top-K documents with their scores.
✓
SearchResult[] returned
Mapped to { txnId, description, status, riskScore, similarity }. React renders similarity score bars and risk/status badges.

Build & Deployment

1
Build React frontend
npm ci && npm run build inside frontend/. Vite outputs the bundle to backend/src/main/resources/static/ (configured via build.outDir in vite.config.js). Spring Boot serves it at /.
Vite 5React 18Tailwind CSS 3
2
Package Spring Boot jar
mvn clean package -DskipTests inside backend/. Maven bundles Java classes, Spring AI dependencies, and the React static files into a single fat jar: backend/target/payflow-ai.jar.
Spring Boot 3.5Spring AI 1.0JDK 21
3
Set AI credentials
cf set-env payflow-ai AI_BASE_URL <endpoint> and cf set-env payflow-ai AI_API_KEY <key>. Never commit the API key to manifest.yml. Models are already set in the manifest (gpt-oss:20b, nomic-embed-text-v2-moe).
cf set-envNo secrets in git
4
cf push
CF CLI reads manifest.yml, uploads payflow-ai.jar, stages with java_buildpack_offline (JDK 21), assigns PORT, and starts the app. No service bindings needed — no external DB.
java_buildpack_offlineJDK 211G memory
5
Startup — vector store seeded
On first request after startup, SemanticSearchService@PostConstruct runs: 20 documents are embedded via nomic-embed-text-v2-moe and loaded into the in-memory SimpleVectorStore. Takes ~2–5s depending on embedding latency.
@PostConstructEmbeddingModelSimpleVectorStore
✓
App is live
React SPA at /. Chat at POST /api/chat. Search at GET /api/search. Health at /actuator/health. No CORS — frontend and backend share one CF route.
Single routeNo CORS
